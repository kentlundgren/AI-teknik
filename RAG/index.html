RAG – Retrieval-Augmented Generation

En pedagogisk och kontrollerbar genomgång av Classic RAG och Advanced RAG. Denna sida förklarar vad som finns i RAG/-mappen och hur delarna hänger ihop.

Vad handlar RAG-mappen om?

RAG (Retrieval-Augmented Generation) innebär att man hämtar relevant information och låter en språkmodell generera svar baserat på den hämtade informationen. Det ger mer korrekta, källförankrade svar och minskar risken för hallucinationer.

Målet med den här delen av projektet är att kunna förstå och styra både klassisk och avancerad RAG – teoretiskt och praktiskt. Dokumenten är skrivna så att de är portabla mellan olika AI-verktyg (Claude, Cursor, LangChain m.fl.).

Classic RAG vs Advanced RAG

Classic RAG – den enkla pipelinen

Fråga → Query Embedding → Similarity Search → Top-K → LLM → Svar

Frågan görs om till en vektor (embedding), man söker de mest lika textbitarna i en vektordatabas och skickar de bästa träffarna till modellen.

Använd när: frågan är enkel, datamängden liten, eller som baseline.

Analogi: som att slå upp i en bok med bara innehållsförteckningen till hjälp.

Advanced RAG – den utökade pipelinen

Fråga → Hybrid Retrieval (Dense + Sparse) → Metadata Filtering
      → Reranking → Relevance Filtering → Context Fusion → LLM → Svar
  • Hybrid Retrieval: kombinerar semantisk (dense) och nyckelord (sparse/BM25).
  • Metadata Filtering: filtrerar på källa, datum, version m.m.
  • Reranking: omvärderar träffarna för bättre ordning.
  • Relevance Filtering: tar bort irrelevanta chunks.
  • Context Fusion: syntetiserar ett sammanhängande svar med källhänvisningar.

Använd när: hög kvalitet krävs, tekniska domäner, stora datamängder.

Analogi: som en smart assistent som söker, sorterar, filtrerar och sammanfattar åt dig.

Production/Advanced RAG = Retrieval + Ranking + Filtering + Fusion
